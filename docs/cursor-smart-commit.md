# smart-commit

Используй инструмент **smart-commit** (пакет npm `smart-commit-ai`, алиасы CLI: `smc`, `smit`) для создания conventional commit сообщений с помощью AI.

## Описание

Smart Commit Tool — инструмент для автоматической генерации сообщений в формате Conventional Commits на основе изменений в репозитории. Он анализирует изменения, формирует структурированное сообщение и при обычном запуске создаёт коммит (`git commit --no-verify`). Если настроен remote, выполняется `git push`.

Если есть незакоммиченные незастейдженные изменения, перед генерацией выполняется `git add .` — при необходимости сначала застейджи только нужные файлы вручную (`git add`).

## Основные опции

- `--dry-run` — Показать, что будет закоммичено, без внесения изменений
- `--generate-only` — Только сгенерировать commit сообщение (вывод в терминал), без коммита
- `-m, --message <message>` — Использовать кастомное commit сообщение вместо генерации AI
- `-v, --verbose` — Подробный вывод
- `--force` — Принудительно выполнить commit при кастомном сообщении (`-m`), если валидация не прошла

## Команды

- `config [options]` — Управление конфигурацией (`--global`, `--set key=value`, `--get key`, `--list`; вложенные ключи, например `apiKeys.openai`)
- `models` — Управление AI моделями: подкоманды `list` (`--all`), `refresh`, `clear-cache`
- `setup` — Начальная настройка и проверка (ключи API, git-репозиторий)
- `generate-config` — Генерация `.smart-commit.json` для текущего проекта (в т.ч. черновик настроек деплоя)
- `deploy` — Деплой по SSH: по умолчанию **умный** режим (анализ `git diff`, только нужные удалённые команды; опционально `localCommands` на локальной машине до SSH)
- `deploy --full` — Полный прогон всех категорий команд из конфига без smart-анализа
- `deploy-smart` — Устарело: то же, что `deploy`; выводит предупреждение (используй `deploy`)
- `install-hooks` — Установка Git hooks (commit-msg: валидация и улучшение сообщения через AI)
- `uninstall-hooks` — Удаление Git hooks
- `hook <hook-type> <file>` — Внутренняя команда для Git hooks (например `commit-msg`)

## Использование

Выполни `smart-commit` в терминале из корня git-репозитория. После при необходимости застейджь изменения (`git add`) — инструмент сгенерирует conventional commit и создаст коммит (и push при наличии remote).

Если нужно только сгенерировать сообщение без коммита: `smart-commit --generate-only`.

Для просмотра того, что будет закоммичено: `smart-commit --dry-run`.

Обход hook при ручном `git commit`: `git commit --no-verify` (вызов `smart-commit` для коммита уже использует `--no-verify`).
